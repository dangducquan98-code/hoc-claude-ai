Bài 03

System prompt & vai trò người dùng

Khái niệm 30 phút

Mục tiêu bài học

  • Hiểu sự khác biệt giữa system prompt, user message và assistant message
  • Biết cách dùng system prompt để "đặt tính cách" cho Claude
  • Nắm được khi nào nên và không nên dùng system prompt
  • Thực hành viết system prompt hiệu quả

3 loại message trong cuộc trò chuyện

Mỗi cuộc trò chuyện với Claude có 3 loại tin nhắn:

  • System prompt: Hướng dẫn "ngầm" mà Claude luôn tuân theo trong suốt cuộc trò chuyện. Người dùng cuối thường không thấy nội dung này.
  • User message: Tin nhắn của bạn — câu hỏi, yêu cầu, dữ liệu bạn gửi.
  • Assistant message: Câu trả lời của Claude.
Hình dung dễ hiểu
Nếu Claude là một nhân viên mới, thì:
System prompt = bản mô tả công việc (JD) mà bạn giao từ đầu
User message = từng yêu cầu cụ thể hàng ngày
Assistant message = kết quả nhân viên trả về

System prompt hoạt động thế nào?

System prompt được đặt trước tất cả các tin nhắn khác. Claude đọc nó đầu tiên và coi đó là "luật chơi" cho toàn bộ cuộc trò chuyện. Điều này có nghĩa:

  • Claude sẽ tuân theo system prompt trong mọi câu trả lời
  • Bạn không cần nhắc lại yêu cầu mỗi lần gửi tin nhắn
  • System prompt ảnh hưởng đến giọng văn, phong cách, kiến thức và giới hạn của Claude

Cách đặt system prompt trên Claude.ai

Trên claude.ai, bạn có thể đặt system prompt thông qua tính năng Projects:

  1. Tạo một Project mới (hoặc mở project có sẵn)
  2. Trong phần "Project instructions", nhập system prompt
  3. Mọi cuộc trò chuyện trong project đó sẽ tự động sử dụng system prompt này
Lưu ý
Nếu dùng Claude qua API, system prompt được truyền qua tham số system trong mỗi request. Nếu dùng trên claude.ai ở chế độ chat thông thường (không trong Project), bạn có thể đặt hướng dẫn ngay ở tin nhắn đầu tiên — tuy không phải system prompt thật sự nhưng hiệu quả tương đương cho hầu hết trường hợp.

Ví dụ system prompt thực tế

Trợ lý viết content tiếng Việt

Bạn là trợ lý viết content chuyên nghiệp cho thị trường Việt Nam.

Quy tắc:
- Luôn viết tiếng Việt tự nhiên, tránh dịch máy
- Giọng văn: thân thiện, dễ hiểu, không quá trang trọng
- Mỗi đoạn văn không quá 3-4 câu
- Luôn đề xuất tiêu đề hấp dẫn trước khi viết nội dung
- Khi được hỏi về chủ đề bạn không chắc, hãy nói rõ thay vì bịa

Gia sư toán cho học sinh cấp 3

Bạn là gia sư toán cho học sinh cấp 3 Việt Nam.

Phương pháp:
- Luôn hỏi học sinh đã hiểu đến đâu trước khi giải thích
- Giải thích từng bước, không nhảy cóc
- Dùng ví dụ từ cuộc sống để minh họa khái niệm trừu tượng
- Khi học sinh sai, không chê — hướng dẫn tìm lỗi sai
- Cuối mỗi bài, đặt 1-2 câu hỏi kiểm tra hiểu bài

Giới hạn:
- Chỉ hỗ trợ toán cấp 3 (lớp 10-12)
- Không làm bài thay — chỉ hướng dẫn cách làm

Trợ lý phân tích dữ liệu

Bạn là data analyst chuyên phân tích dữ liệu kinh doanh.

Khi nhận dữ liệu:
1. Tóm tắt những điểm nổi bật trước
2. Chỉ ra xu hướng và anomaly
3. Đề xuất hành động cụ thể

Format output:
- Luôn dùng bảng khi so sánh số liệu
- Dùng bullet points cho insights
- Kết thúc bằng phần "Đề xuất tiếp theo"

Nếu dữ liệu không đủ để kết luận, hãy nói rõ cần thêm gì.

System prompt tốt vs tệ

Tệ — Quá mơ hồ
Hãy trả lời tốt và hữu ích.

→ Claude đã được thiết kế để làm điều này rồi. System prompt này không thêm giá trị gì.
Tệ — Quá dài và mâu thuẫn
Luôn trả lời ngắn gọn. Giải thích chi tiết mọi khái niệm. Dùng nhiều ví dụ nhưng giữ câu trả lời dưới 100 từ.

→ Các yêu cầu mâu thuẫn khiến Claude bối rối.
Tốt — Rõ ràng, có cấu trúc
System prompt tốt nên có:
  • Vai trò: Claude là ai trong cuộc trò chuyện này
  • Quy tắc: Những gì nên và không nên làm
  • Phong cách: Giọng văn, format output
  • Giới hạn: Phạm vi Claude nên hoạt động

Bài tập thực hành

  1. Viết system prompt cho một trợ lý viết email công việc hàng ngày. Thử trong Project trên claude.ai.
  2. Viết system prompt cho một "đầu bếp AI" chỉ gợi ý món ăn Việt Nam với nguyên liệu bạn có sẵn.
  3. Tạo 2 cuộc trò chuyện riêng biệt: một có system prompt, một không — gửi cùng câu hỏi và so sánh kết quả.

Tổng kết

  • System prompt = "luật chơi" áp dụng cho toàn bộ cuộc trò chuyện
  • Đặt qua Project instructions trên claude.ai, hoặc tham số system qua API
  • Nên rõ ràng, có cấu trúc: vai trò → quy tắc → phong cách → giới hạn
  • Tránh mơ hồ hoặc mâu thuẫn — Claude sẽ cố gắng tuân theo mọi thứ bạn viết
