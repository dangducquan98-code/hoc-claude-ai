Artifacts — tạo output có thể tương tác

Thực hành 60 phút

Mục tiêu bài học

  • Hiểu Artifacts là gì và tại sao chúng khác biệt so với câu trả lời thông thường
  • Biết các loại Artifact: HTML, code, SVG, React component, tài liệu Markdown
  • Phân biệt khi nào Claude tạo Artifact và khi nào trả lời inline
  • Thực hành tạo 5 loại Artifact khác nhau
  • Nắm mẹo để Claude tạo Artifact chất lượng cao

Artifacts là gì?

Artifacts là tính năng trên claude.ai cho phép Claude tạo ra nội dung độc lập hiển thị trong một panel riêng bên phải màn hình. Thay vì hiện mọi thứ trong khung chat, Claude "đẩy" nội dung có cấu trúc sang một cửa sổ riêng — nơi bạn có thể xem, tương tác, copy, và tải về.

Hình dung dễ hiểu

Chat thường giống như Claude nói cho bạn nghe câu trả lời.

Artifacts giống như Claude đưa cho bạn một tài liệu hoặc sản phẩm — bạn có thể mở ra, xem, sửa, tải về, thậm chí chạy ngay trên trình duyệt.

Ví dụ trực quan

Khi bạn nói: "Tạo cho tôi một trang web giới thiệu bản thân đơn giản"

  • Không có Artifacts: Claude hiển thị code HTML trong khung chat — bạn phải copy, paste vào file, rồi mở trên trình duyệt
  • Có Artifacts: Claude tạo một Artifact hiển thị ngay trang web hoàn chỉnh bên phải — bạn thấy giao diện ngay lập tức, có thể click thử, và tải file HTML về

Các loại Artifact

1. HTML — Trang web tương tác

Đây là loại Artifact phổ biến và mạnh nhất. Claude tạo ra trang web hoàn chỉnh (HTML + CSS + JavaScript) chạy trực tiếp trên trình duyệt.

Phù hợp cho:

  • Landing page, portfolio cá nhân
  • Công cụ tính toán (máy tính BMI, tính lãi suất, tính thuế)
  • Game đơn giản (quiz, flashcard, Tic-tac-toe)
  • Dashboard, biểu đồ trực quan
  • Form thu thập thông tin
Ví dụ prompt

"Tạo một công cụ tính thuế thu nhập cá nhân Việt Nam 2026. Giao diện tiếng Việt, cho nhập thu nhập hàng tháng, số người phụ thuộc, tự động tính thuế phải nộp. Giao diện đẹp, màu xanh dương."

2. Code — Mã nguồn

Claude tạo file code với syntax highlighting. Bạn có thể copy toàn bộ code chỉ bằng một click.

Phù hợp cho:

  • Script Python, JavaScript, hoặc bất kỳ ngôn ngữ nào
  • File cấu hình (JSON, YAML, .env)
  • SQL queries
  • Đoạn code dài mà bạn cần copy nguyên vẹn
Ví dụ prompt

"Viết script Python đọc file Excel bán hàng, tính tổng doanh thu theo tháng, và xuất ra biểu đồ cột. Dùng pandas và matplotlib."

3. SVG — Hình vẽ vector

Claude có thể tạo hình ảnh vector (SVG) — từ icon đơn giản đến sơ đồ phức tạp.

Phù hợp cho:

  • Logo đơn giản, icon
  • Sơ đồ quy trình, flowchart
  • Infographic đơn giản
  • Hình minh họa cho bài thuyết trình
Ví dụ prompt

"Vẽ sơ đồ quy trình tuyển dụng gồm 6 bước: Đăng tin → Sàng lọc CV → Phỏng vấn vòng 1 → Phỏng vấn vòng 2 → Offer → Onboarding. Dùng SVG, phong cách chuyên nghiệp, màu xanh lá."

4. React Component — Ứng dụng web nâng cao

Đối với ứng dụng phức tạp hơn HTML thuần, Claude có thể tạo React component với thư viện hỗ trợ (ví dụ: recharts cho biểu đồ, lucide-react cho icon).

Phù hợp cho:

  • Dashboard dữ liệu với biểu đồ tương tác
  • Ứng dụng có state phức tạp (quản lý todo, kanban board)
  • Công cụ có nhiều tính năng và tương tác phức tạp
Ví dụ prompt

"Tạo dashboard theo dõi chi tiêu cá nhân. Có thể nhập khoản chi (tên, số tiền, danh mục). Hiển thị biểu đồ tròn phân bổ theo danh mục và biểu đồ đường theo thời gian. Giao diện tiếng Việt."

5. Tài liệu Markdown

Đối với nội dung dạng văn bản dài có cấu trúc, Claude tạo Artifact dạng tài liệu — hiển thị đẹp, dễ đọc, dễ tải về.

Phù hợp cho:

  • Báo cáo, bài phân tích dài
  • Kế hoạch dự án, proposal
  • Hướng dẫn sử dụng, tài liệu kỹ thuật
  • Nội dung bài viết blog dài
Ví dụ prompt

"Viết kế hoạch marketing 3 tháng cho quán cà phê mới mở tại quận 3, TP.HCM. Bao gồm: phân tích đối thủ, chiến lược online/offline, ngân sách dự kiến, timeline từng tuần, KPI cần đo."

Khi nào Claude tạo Artifact vs trả lời inline?

Claude không phải lúc nào cũng tạo Artifact. Dưới đây là quy tắc chung:

Claude SẼ tạo Artifact khi:

  • Nội dung đủ dài và có cấu trúc (thường trên 15-20 dòng)
  • Nội dung là code, HTML, hoặc SVG hoàn chỉnh
  • Bạn yêu cầu tạo "trang web", "ứng dụng", "tool", "tài liệu"
  • Nội dung có thể đứng độc lập (tự đủ ý nghĩa khi tách khỏi cuộc chat)

Claude sẽ KHÔNG tạo Artifact khi:

  • Câu trả lời ngắn, chỉ vài dòng
  • Đoạn code nhỏ minh họa trong giải thích
  • Câu trả lời gắn chặt với ngữ cảnh cuộc chat (không có ý nghĩa khi tách ra)
  • Bạn chỉ hỏi câu hỏi thông thường
Mẹo

Nếu bạn muốn Claude tạo Artifact, hãy dùng các từ khóa gợi ý:

  • "Tạo một trang web..."
  • "Viết một ứng dụng..."
  • "Tạo tool tính toán..."
  • "Viết tài liệu hoàn chỉnh về..."

Ngược lại, nếu không muốn Artifact: "Giải thích ngắn gọn trong chat..." hoặc "Trả lời trực tiếp, không cần tạo artifact."

Thực hành: Tạo 5 Artifact khác nhau

Hãy mở claude.ai và thực hành tạo từng loại Artifact dưới đây. Mỗi Artifact, hãy thử tương tác, chỉnh sửa, và tải về.

Thực hành 1: Trang web cá nhân (HTML)

Copy và gửi prompt này cho Claude:

Tạo một trang web portfolio cá nhân cho một lập trình viên Việt Nam. Gồm:
- Header với tên và tiêu đề nghề nghiệp
- Phần "Về tôi" với ảnh placeholder
- Phần "Kỹ năng" hiển thị dạng progress bar
- Phần "Dự án" với 3 card sản phẩm
- Footer với link social media
Thiết kế tối giản, hiện đại, responsive. Dùng font system.
Nội dung mẫu bằng tiếng Việt.

Sau khi nhận Artifact: Thử resize cửa sổ để kiểm tra responsive. Click các link. Tải HTML về và mở trên máy tính.

Thực hành 2: Công cụ tính toán (HTML + JS)

Tạo công cụ tính lãi suất tiết kiệm ngân hàng. Cho nhập:
- Số tiền gửi (VND)
- Kỳ hạn (1, 3, 6, 12, 24 tháng)
- Lãi suất (%/năm)

Hiển thị kết quả:
- Tiền lãi nhận được
- Tổng tiền nhận khi đáo hạn
- Bảng so sánh nếu gửi ở các kỳ hạn khác nhau

Giao diện tiếng Việt, format số tiền có dấu chấm phân cách hàng nghìn.
Thiết kế chuyên nghiệp, dễ dùng trên điện thoại.

Sau khi nhận Artifact: Nhập số tiền thử và kiểm tra kết quả bằng máy tính. Thử trên điện thoại.

Thực hành 3: Sơ đồ quy trình (SVG)

Vẽ sơ đồ quy trình xử lý đơn hàng online gồm các bước:
1. Khách đặt hàng trên website
2. Hệ thống xác nhận đơn hàng
3. Kiểm tra tồn kho (nếu hết hàng → thông báo khách, kết thúc)
4. Đóng gói
5. Giao cho đơn vị vận chuyển
6. Giao hàng thành công
7. Khách xác nhận nhận hàng

Dùng SVG. Hình chữ nhật bo tròn cho bước thường, hình thoi cho điều kiện.
Phong cách: flat design, màu xanh dương và xám.

Sau khi nhận Artifact: Xem sơ đồ có đúng logic không. Thử yêu cầu Claude sửa màu hoặc thêm bước.

Thực hành 4: Dashboard dữ liệu (React)

Tạo dashboard quản lý quán cà phê với dữ liệu mẫu, gồm:
- Tổng doanh thu hôm nay, tuần này, tháng này (hiển thị số lớn)
- Biểu đồ cột: doanh thu 7 ngày gần nhất
- Biểu đồ tròn: tỷ lệ doanh thu theo loại đồ uống (cà phê, trà, sinh tố, khác)
- Bảng 5 sản phẩm bán chạy nhất

Dùng dữ liệu mẫu thực tế (đơn vị VND).
Giao diện tiếng Việt, phong cách hiện đại, nền tối (dark mode).

Sau khi nhận Artifact: Hover lên biểu đồ xem có tooltip không. Thử yêu cầu Claude đổi sang light mode hoặc thêm biểu đồ mới.

Thực hành 5: Tài liệu có cấu trúc (Markdown)

Viết tài liệu hướng dẫn onboarding cho nhân viên mới tại một công ty
phần mềm Việt Nam. Gồm:

1. Chào mừng và giới thiệu công ty
2. Tuần đầu tiên — checklist việc cần làm
3. Công cụ và tài khoản cần setup (email, Slack, Jira, Git)
4. Quy trình làm việc hàng ngày (standup, code review, deploy)
5. Văn hóa công ty và quy tắc ứng xử
6. Danh sách người liên hệ khi cần hỗ trợ
7. FAQ cho nhân viên mới

Nội dung chi tiết, thực tế, phù hợp với công ty phần mềm
tầm 50-100 người ở Việt Nam.

Sau khi nhận Artifact: Đọc qua nội dung, kiểm tra cấu trúc. Tải về dùng làm template thực tế.

Chỉnh sửa và lặp lại Artifact

Một trong những điểm mạnh của Artifacts là bạn có thể yêu cầu Claude chỉnh sửa ngay trên Artifact đã tạo:

  • "Đổi màu nền thành xanh đậm"
  • "Thêm cột 'Ghi chú' vào bảng"
  • "Thay font chữ thành Roboto"
  • "Thêm animation khi hover vào button"
  • "Sửa phần tính thuế — quên trừ giảm trừ gia cảnh"

Claude sẽ cập nhật Artifact hiện tại thay vì tạo mới — bạn thấy thay đổi ngay lập tức.

Lưu ý

Bạn có thể yêu cầu chỉnh sửa nhiều lần. Artifact cũng có version history — bạn có thể quay lại phiên bản trước nếu chỉnh sửa không ưng ý.

Mẹo để Claude tạo Artifact tốt nhất

Mẹo
  • Mô tả càng cụ thể càng tốt: Thay vì "tạo trang web đẹp", hãy nói "tạo trang web tối giản, nền trắng, font sans-serif, màu chủ đạo xanh dương #2563EB"
  • Nêu rõ đối tượng sử dụng: "Cho người dùng Việt Nam, hiển thị tiền VND, tên tiếng Việt có dấu"
  • Nêu rõ thiết bị: "Cần responsive, hoạt động tốt trên điện thoại" hoặc "chỉ cần chạy trên desktop"
  • Cung cấp dữ liệu mẫu: Nếu có, đưa dữ liệu thật hoặc gần thật — Claude sẽ tạo Artifact chân thực hơn
  • Chia nhỏ yêu cầu phức tạp: Nếu muốn ứng dụng lớn, hãy bắt đầu với phiên bản đơn giản rồi thêm tính năng dần
  • Nói rõ phong cách: "Phong cách Material Design", "giống giao diện Notion", "tối giản kiểu Apple" — giúp Claude hiểu aesthetic bạn muốn

Giới hạn của Artifacts

Artifacts rất mạnh nhưng có giới hạn:

  • Không kết nối internet: Artifact HTML/React chạy sandboxed — không gọi được API bên ngoài, không tải hình từ URL bên ngoài
  • Không lưu dữ liệu: Khi refresh, dữ liệu bạn nhập vào Artifact sẽ mất (không có database)
  • Độ phức tạp có giới hạn: Artifact phù hợp cho prototype và công cụ đơn giản — không thay thế được ứng dụng production
  • Một số thư viện không hỗ trợ: Trong React Artifact, chỉ một số thư viện phổ biến được hỗ trợ sẵn
Cảnh báo

Đừng kỳ vọng Artifact thay thế developer thực thụ. Artifact tuyệt vời cho: prototype nhanh, công cụ nội bộ đơn giản, demo ý tưởng, và học lập trình. Nếu cần ứng dụng production phức tạp, hãy dùng Artifact làm bản nháp rồi nhờ developer phát triển tiếp.

Ứng dụng thực tế cho người Việt

Dưới đây là các ý tưởng Artifact hữu ích mà bạn có thể tạo ngay:

Cho công việc

  • Máy tính thuế TNCN, thuế GTGT
  • Template báo cáo tuần hoặc tháng
  • Dashboard KPI đội nhóm
  • Công cụ chuyển đổi đơn vị, tiền tệ
  • Trang landing page cho sản phẩm mới (demo trước khi code thật)

Cho học tập

  • Flashcard ôn từ vựng tiếng Anh
  • Quiz trắc nghiệm cho bất kỳ môn nào
  • Sơ đồ tư duy (mind map) cho bài học
  • Bảng so sánh các khái niệm

Cho cá nhân

  • Công cụ theo dõi chi tiêu
  • Lịch lên kế hoạch du lịch
  • Thiệp mời sinh nhật, thiệp Tết
  • CV hoặc portfolio cá nhân đơn giản
  • Mini game giải trí

Bài tập thực hành

  1. Tạo 5 Artifact: Hoàn thành tất cả 5 bài thực hành phía trên. Mỗi Artifact, thử chỉnh sửa ít nhất 2 lần.
  2. Sáng tạo Artifact riêng: Nghĩ ra một công cụ hoặc trang web mà bạn thực sự cần trong công việc hoặc cuộc sống. Mô tả chi tiết cho Claude và tạo Artifact.
  3. Thử giới hạn: Yêu cầu Claude tạo một Artifact phức tạp (ví dụ: ứng dụng Kanban board có drag-and-drop). Quan sát Claude xử lý đến đâu và giới hạn ở đâu.
  4. Tải và sử dụng: Tải ít nhất 1 Artifact HTML về máy tính, mở trong trình duyệt, và thử dùng như một công cụ thực sự.

Tổng kết

  • Artifacts là tính năng Claude tạo nội dung độc lập trong panel riêng — có thể tương tác, copy, tải về
  • 5 loại chính: HTML (trang web), Code (mã nguồn), SVG (hình vector), React (ứng dụng nâng cao), Markdown (tài liệu)
  • Claude tự quyết định tạo Artifact khi nội dung đủ dài, có cấu trúc, và có thể đứng độc lập
  • Có thể chỉnh sửa lặp lại nhiều lần — Claude cập nhật trực tiếp trên Artifact hiện tại
  • Mô tả càng cụ thể (màu sắc, bố cục, đối tượng, dữ liệu mẫu) thì Artifact càng đúng ý
  • Artifacts phù hợp cho prototype, công cụ đơn giản, demo ý tưởng — không thay thế ứng dụng production
