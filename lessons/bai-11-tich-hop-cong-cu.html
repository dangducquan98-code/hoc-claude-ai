Tích hợp công cụ — Google Drive, Notion, v.v.

Thực hành 60 phút

Mục tiêu bài học

  • Hiểu MCP (Model Context Protocol) — cách Claude kết nối với các công cụ bên ngoài
  • Biết cách tích hợp Claude với Google Drive, Notion, và các ứng dụng hàng ngày
  • Thực hành quy trình: đọc dữ liệu từ công cụ → phân tích bằng Claude → tạo output
  • Xây dựng workflow tự động hóa thực tế cho công việc

Tại sao cần tích hợp công cụ?

Ở các bài trước, bạn đã dùng Claude chủ yếu trong giao diện chat: gõ prompt, nhận câu trả lời. Nhưng trong công việc thực tế, dữ liệu của bạn nằm ở nhiều nơi khác nhau:

  • Tài liệu trên Google Drive
  • Ghi chú và database trên Notion
  • Code trên GitHub
  • Dữ liệu trên Google Sheets
  • Email trên Gmail

Nếu mỗi lần cần Claude phân tích, bạn phải copy-paste thủ công thì rất mất thời gian. Tích hợp công cụ cho phép Claude trực tiếp đọc và ghi dữ liệu từ các ứng dụng này.

MCP — Model Context Protocol là gì?

MCP (Model Context Protocol) là tiêu chuẩn mở do Anthropic phát triển, cho phép các ứng dụng AI kết nối với nguồn dữ liệu và công cụ bên ngoài. Hãy hình dung MCP như ổ cắm USB chuẩn — bất kỳ thiết bị nào hỗ trợ USB đều có thể kết nối, và MCP làm điều tương tự cho AI.

Lưu ý
MCP là tiêu chuẩn mở: Không chỉ Claude, mà các AI khác cũng có thể hỗ trợ MCP. Anthropic xây dựng MCP như một tiêu chuẩn chung cho toàn ngành, giúp bạn không bị khóa vào một nền tảng.

MCP hoạt động như thế nào?

Kiến trúc MCP gồm 3 phần:

  1. MCP Server: Chương trình kết nối với một dịch vụ cụ thể (Google Drive, Notion, v.v.). Server này "biết" cách đọc/ghi dữ liệu từ dịch vụ đó.
  2. MCP Client: Ứng dụng AI (Claude Desktop, Claude Code) giao tiếp với MCP Server để lấy dữ liệu hoặc thực hiện hành động.
  3. Giao thức (Protocol): Quy chuẩn cách Client và Server nói chuyện với nhau — đảm bảo mọi thứ hoạt động trơn tru.
Ví dụ

Khi bạn yêu cầu Claude "Đọc file báo cáo tuần trên Google Drive và tóm tắt":

  1. Claude (MCP Client) gửi yêu cầu đến Google Drive MCP Server
  2. MCP Server kết nối với Google Drive API, lấy nội dung file
  3. Nội dung file được gửi lại cho Claude
  4. Claude phân tích và tóm tắt cho bạn

Các tích hợp phổ biến trên Claude.ai

Trên giao diện web Claude.ai, bạn có thể kết nối trực tiếp với một số dịch vụ phổ biến:

Google Drive

  • Đọc: Google Docs, Sheets, Slides, PDF
  • Cách dùng: Trong cuộc trò chuyện, nhấn biểu tượng đính kèm → chọn Google Drive → chọn file
  • Ứng dụng: Tóm tắt tài liệu dài, phân tích dữ liệu bảng tính, trích xuất thông tin từ slide

Google Docs kết hợp với Claude

  • Đọc và phân tích: Claude có thể đọc nội dung Google Docs và phân tích, góp ý, tóm tắt
  • Ứng dụng: Review tài liệu, trích xuất action items từ biên bản họp, so sánh nhiều phiên bản

Notion (qua MCP)

  • Đọc: Pages, databases, comments
  • Ghi: Tạo page mới, cập nhật database
  • Ứng dụng: Đọc task board → tóm tắt tiến độ, phân tích database khách hàng, tạo trang wiki từ ghi chú
Mẹo
Khi kết nối với các dịch vụ, Claude sẽ hỏi quyền truy cập. Hãy chỉ cấp quyền tối thiểu cần thiết — ví dụ nếu chỉ cần đọc thì không cần cấp quyền ghi. Đây là nguyên tắc bảo mật quan trọng.

Thiết lập tích hợp từng bước

Bước 1: Kết nối Google Drive với Claude

  1. Mở claude.ai và bắt đầu cuộc trò chuyện mới
  2. Nhấn biểu tượng đính kèm (attach) trong ô nhập tin nhắn
  3. Chọn Google Drive
  4. Đăng nhập tài khoản Google và cấp quyền truy cập
  5. Duyệt và chọn file bạn muốn Claude đọc

Bước 2: Sử dụng trong Project

Bạn có thể thêm file từ Google Drive vào Project Knowledge để Claude luôn có sẵn context:

  1. Vào Project → Add Content
  2. Chọn nguồn Google Drive
  3. Chọn các file cần thêm
  4. File sẽ được đồng bộ — khi bạn cập nhật file trên Drive, Claude cũng có dữ liệu mới

Bước 3: Thiết lập MCP cho ứng dụng nâng cao

Nếu bạn dùng Claude Desktop app, bạn có thể cài đặt thêm các MCP server:

// Ví dụ cấu hình MCP trong Claude Desktop
// File: claude_desktop_config.json

{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-notion"],
      "env": {
        "NOTION_API_KEY": "your-notion-api-key"
      }
    },
    "google-drive": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-google-drive"]
    }
  }
}
Cảnh báo
Cấu hình MCP server đòi hỏi kiến thức kỹ thuật cơ bản (biết dùng terminal, cài Node.js). Nếu bạn không quen, hãy bắt đầu với các tích hợp có sẵn trên Claude.ai trước — chúng đủ mạnh cho hầu hết nhu cầu.

Workflow thực tế: Từ Notion đến báo cáo

Hãy xem một workflow hoàn chỉnh — từ lúc đọc dữ liệu đến tạo output:

Tình huống

Bạn quản lý dự án trên Notion. Mỗi tuần, bạn cần tóm tắt tiến độ cho sếp. Thay vì mở Notion → đọc → tự viết, hãy để Claude giúp.

Bước 1: Đọc dữ liệu từ Notion

Prompt: "Đọc database 'Sprint Board' trên Notion. Liệt kê các task theo trạng thái: Done, In Progress, Blocked."

Bước 2: Phân tích

Prompt: "Dựa trên dữ liệu vừa đọc: (1) Tính % hoàn thành sprint, (2) Liệt kê task bị blocked và lý do, (3) So sánh tiến độ với sprint trước."

Bước 3: Tạo báo cáo

Prompt: "Viết email báo cáo tuần cho anh Minh (PM Director) dựa trên phân tích trên. Tone chuyên nghiệp, ngắn gọn, nêu rõ risks."

Mẹo
Kết hợp workflow này với Custom Skills (Bài 9): viết Skill "Báo cáo tuần" trong Project Instructions, thì mỗi tuần bạn chỉ cần nói "Tạo báo cáo tuần" — Claude tự biết đọc Notion, phân tích, và viết email theo đúng format.

Workflow thực tế: Google Sheets → Phân tích → Slide

Một ví dụ khác cho team kinh doanh:

Bước 1: Đọc dữ liệu bán hàng từ Google Sheets

"Đọc file 'Sales Q1 2026' trên Google Drive. Đây là dữ liệu doanh số theo tháng của 5 khu vực."

Bước 2: Phân tích chuyên sâu

"Phân tích: Khu vực nào tăng trưởng tốt nhất? Khu vực nào cần chú ý? Có pattern gì đặc biệt? So sánh với target."

Bước 3: Tạo nội dung thuyết trình

"Tạo outline cho bài thuyết trình 10 slide về kết quả Q1, bao gồm: highlight, phân tích từng khu vực, và đề xuất cho Q2."

Các tích hợp hữu ích khác

GitHub

Kết nối để Claude đọc code, review pull request, tạo documentation từ codebase.

Slack (qua MCP)

Claude có thể đọc tin nhắn kênh, tóm tắt thảo luận, trả lời câu hỏi dựa trên lịch sử chat.

Zapier / Make (qua API)

Kết nối Claude với hàng nghìn ứng dụng thông qua các nền tảng automation. Ví dụ: khi có email mới → Claude phân tích → tạo task trên Notion.

Lưu ý
Hệ sinh thái MCP đang phát triển rất nhanh. Cộng đồng liên tục tạo thêm các MCP server mới. Bạn có thể theo dõi tại github.com/modelcontextprotocol để cập nhật các tích hợp mới nhất.

Bảo mật khi tích hợp

Khi cho Claude truy cập dữ liệu công ty, hãy lưu ý:

  • Nguyên tắc quyền tối thiểu: Chỉ cấp quyền đọc nếu không cần ghi. Chỉ chia sẻ folder cần thiết, không chia sẻ toàn bộ Drive.
  • Không chia sẻ dữ liệu nhạy cảm: Tránh cho Claude truy cập file chứa mật khẩu, thông tin thẻ tín dụng, dữ liệu cá nhân nhạy cảm của khách hàng.
  • Kiểm tra chính sách công ty: Một số công ty có quy định về việc sử dụng AI với dữ liệu nội bộ. Hãy kiểm tra trước khi kết nối.
  • Review quyền định kỳ: Mỗi tháng, kiểm tra lại các kết nối đã cấp quyền và thu hồi những quyền không còn cần.

Bài tập thực hành

  1. Kết nối Google Drive: Kết nối Claude với Google Drive. Upload hoặc chọn một tài liệu có sẵn. Yêu cầu Claude tóm tắt, rồi phân tích điểm mạnh/yếu của tài liệu.
  2. Workflow hoàn chỉnh: Thực hiện một workflow 3 bước: (1) cho Claude đọc dữ liệu từ một nguồn (Drive/upload file), (2) yêu cầu phân tích, (3) tạo output có cấu trúc (báo cáo, email, hoặc outline thuyết trình).
  3. Tích hợp vào Project: Tạo một Project, thêm file từ Google Drive vào Knowledge. Viết System Prompt bao gồm Skill xử lý dữ liệu từ file đó. Kiểm tra bằng cách gửi yêu cầu.
  4. Lên kế hoạch tích hợp: Liệt kê 5 tác vụ lặp đi lặp lại trong công việc của bạn mà có thể tự động hóa bằng cách kết nối Claude với các công cụ. Với mỗi tác vụ, mô tả: nguồn dữ liệu → phân tích cần làm → output mong muốn.

Tổng kết

  • MCP (Model Context Protocol) là tiêu chuẩn mở cho phép Claude kết nối với các công cụ bên ngoài như Google Drive, Notion, GitHub
  • Trên Claude.ai, tích hợp Google Drive có sẵn — chỉ cần vài click để kết nối
  • Workflow hiệu quả: đọc dữ liệu từ công cụ → phân tích bằng Claude → tạo output có cấu trúc
  • Kết hợp tích hợp với Projects và Skills để tự động hóa tác vụ lặp lại
  • Luôn tuân thủ nguyên tắc bảo mật: quyền tối thiểu, không chia sẻ dữ liệu nhạy cảm, review định kỳ
